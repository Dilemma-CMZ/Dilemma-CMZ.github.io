2019 Multi-University Training Contest 1 String

发表于 2019-07-22 | 更新于 2019-07-23 | 分类于 Multi-University Training Contest ， 贪心
本文字数： 10k | 阅读时长 ≈ 9 分钟

题目链接 String

Description

你有一个字符串$s$ ，只包含小写字母,你需要从中找到其一个长度为$k$的子序列，使得这个子序列字典序最小，并且我有26个约束。
对于每一个约束，规定了当前字母需要选取的个数的上界和下界。

题解

首先我们考虑，如果没有约束我们会怎么做，因为长度固定，所以显然我们要让前面的字母尽可能小，后面的不需要太在意，所以我只需要不停地去找剩余序列中最小的那个字母，并且判断一下如果取了当前这个字母，这个字母之后的字母的个数再加上之前已经取得的字符的个数是够大于等于我要求的len，如果满足即继续找下一位，不满足就去找次小的。

这道题其实一样，还是按位贪心，只不过改变了一下判断可行的条件.
可行的条件：

1.对于每一个字母x，如果当前已经取得的x的个数加上这一个位置之后的x个数小于$l_i$就不合法.
2.对于所有字母，如果剩下的$\sum_{x=1}^{26} \text{min}{r_x-pick_x,rest_x}$ 小于 $k$减去已经取得的字母数量，则不合法.
3.如果 $sum_{x=1}^{26} l_x$减去已经取得的字母数大于$k$，则不合法.

之后这道题目就和没有约束的做法一样了。
